# Schema: Reglas de Contribución al Repositorio pragma-knowledge

## Propósito
Este documento define las reglas, convenciones y procesos que rigen la contribución al repositorio pragma-knowledge. Es la fuente de verdad sobre cómo se estructura, nombra y gobierna el conocimiento.

## Reglas de Nomenclatura
- Todos los directorios y archivos deben estar en **inglés** y formato **kebab-case**.
- Ejemplo válido: `business-solutions/jtbd/legacy-modernization.md`

## Proceso de Contribución
1. Crear rama desde trunk: `feature/[descripcion-corta]`
2. Realizar cambios siguiendo las plantillas oficiales
3. Abrir Merge Request
4. Peer Review por al menos 1 miembro core del chapter
5. Validación de formato por el Encargado de Validación
6. Merge a trunk
